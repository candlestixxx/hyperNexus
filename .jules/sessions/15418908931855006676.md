# 🏗️ Super AI Plugin (Hypercode/Antigravity) - Comprehensive Architecture & Design Summary

The **Super AI Plugin** (also referred to as **Hypercode** or **Antigravity**) is a high-performance, local-first control plane for AI orchestration, built as a multi-language monorepo. It integrates native Go-based autonomous agent logic with a sophisticated TypeScript/React management hub, aiming for 1:1 tool parity with industry-leading AI assistants like Claude Code, Codex, and Cursor.

## 📂 Monorepo Structure & Ecosystems

### 🔵 Go Ecosystem (`agents/`, `go/internal/`, `foundation/`)
Core orchestration, memory management, and autonomous execution are implemented in Go for performance, concurrency, and native OS integration.
- **`Director`**: The high-level architectural supervisor that plans tasks, delegates to sub-agents, and maintains a structured conversation history.
- **`AutoDrive`**: A state machine implementation of the **Recursive Autonomous Loop** (Roo Code/Cline style). It manages cycles of tool execution natively in Go to minimize latency and context overhead.
- **`DisclosureProxy`**: Implements **Progressive Tool Disclosure**. It protects the LLM context window by stripping massive tool lists and only exposing meta-tools like `search_tools` and `auto_call_tool`.
- **`ShellTranslator`**: A strictly constrained agent role designed for natural language to shell command translation.
- **`ExecuteCodeMode`**: An "escape hatch" allowing models to generate and execute script bundles (TypeScript via `bun`, Bash via `sh`) to perform complex multi-step operations with 94%+ context reduction.
- **`Council`**: Multi-agent consensus logic. Implements a parallel voting system for plan evaluation, using specialized personas (Security Architect, Senior Engineer, DevOps Chief).
- **`SkillRegistry` & `DecisionSystem` (`go/internal/skillregistry`)**: A native Go implementation of the progressive disclosure pattern for skills, including BM25-style ranking and LRU eviction.
- **`foundation/`**: Provides low-level building blocks, including `ContextHarvester`, `GitService`, and the `HypercodeAdapter`.

### 🟢 TypeScript Ecosystem (`packages/`, `apps/`)
TypeScript handles the user-facing dashboard, client extensions, and the initial manager-based control plane.
- **`packages/core`**: The central Hub (Fastify/Socket.io). It employs a **Manager-based architecture** (`AgentManager`, `MemoryManager`, `SkillManager`, `PromptManager`) where specialized services watch the filesystem for real-time configuration and registry updates.
- **`apps/web`**: A Next.js/Vite dashboard providing a "visual observation deck" for system metrics, MCP traffic, provider usage, and the neural transcripts of agent swarms.
- **`packages/cli`**: The `hypercode` / `super-ai` command-line interface.
- **`vscode` & `browser`**: Client extensions that bridge IDE and web environments with the Hub, enabling autonomous context harvesting and local MCP injection into web chats.

## 🧠 Key Architectural Patterns & Decisions

1. **Modular Monolith Strategy**: The project prioritizes logical separation (daemons like `hypercoded`, `hypercodemcpd`) within a single repository to maintain contract stability before physical service extraction.
2. **Dual-Tier Memory (L1/L2)**:
   - **L1 (Scratchpad)**: Ephemeral, lightning-fast memory for the active task history and chain-of-thought.
   - **L2 (Vault)**: Permanent semantic storage using `sqlite-vec` for long-term "lessons learned" and historical recall.
3. **Decision-Based Discovery**: Unlike "raw aggregators," Hypercode is a **Decision System**. It uses semantic search and ranking (BM25/Cosine) to disclose only the most relevant tools or skills on demand, preventing "context window bloat."
4. **Resilient Waterfall Routing**: Inference requests automatically cascade through a prioritized provider chain (NIM → OpenRouter → Local Ollama/LM Studio) to handle 429/5xx errors and quota depletion.
5. **Agent-to-Agent (A2A) Protocol**: A structured communication layer allowing specialized models (Planner, Implementer, Critic) to negotiate tasks via heartbeats and capability reports.
6. **Neural Transcript**: A shared, real-time log of multi-agent collaboration, allowing operators to monitor the "thought process" of an entire swarm.
7. **Handoff Persistence**: State is saved in Markdown "handoff" files and JSON sessions, allowing tasks to be resumed seamlessly across CLI, Web, and IDE clients.
8. **Parity Focus**: The project targets byte-for-byte identical tool signatures for Claude Code, Codex, Gemini CLI, Cursor, and Windsurf to ensure models trained on those signatures behave correctly.

## 🧪 Testing & Reliability Standards
- **Mock-Driven Determinism**: Uses the `mockProvider` pattern in Go to verify complex autonomous and proxy logic without external API costs or latency.
- **Flexible History Assertions**: Tests favor searching history for content/roles rather than hardcoded array indices, ensuring resilience to minor system prompt updates.
- **Environment Adaptability**: Handles sandbox constraints (e.g., restricted Go toolchain downloads) by dynamically aligning `go.mod` version during verification.
- **Runtime-Aware Skips**: Tests for external runtimes (like `bun`) include presence checks and graceful skips to ensure the suite is portable.
